# Этап 0. Выявление требований (Q&A)

Ниже представлены уточняющие вопросы к заказчику (Оле) и ответы, полученные в ходе интервью. Эти ответы легли в основу проектирования MVP.

| # | Вопрос | Ответ заказчика | Влияние на систему |
| :-- | :-- | :-- | :-- |
| 1 | Чем ограничено число мест? | Для новичков (Болдеринг) — 8 мест (безопасность). Для профи (Верёвка) — 16 мест. | Вместимость слота динамическая; сервер — источник истины. |
| 2 | Как работает прокат? | Всего 12 комплектов (скальники + система). Своё снаряжение не расходует фонд, но занимает место в группе. | Два независимых лимита: Места и Снаряжение. |
| 3 | Групповая запись? | Клиент + до 2 гостей. Имена гостей не нужны, только выбор снаряжения. | Метод POST /bookings принимает массив до 3 позиций. |
| 4 | Авторизация? | По номеру телефона через SMS-код. Пароли не нужны. | Интеграция с SMS-шлюзом; сессия по JWT. |
| 5 | Политика отмен? | Бесплатно за 2 часа до старта. Позже — место не освобождается, штраф не взимается. | Статусы: cancelled (ранняя) и late_cancel (поздняя). |
| 6 | Профилактика? | Если зона закрыта, все записи отменяются с уведомлением. | Статус club_cancelled + Push-уведомление. |
| 7 | Оплата? | Наличные или перевод на берегу. Онлайн-оплаты в MVP нет. | В API передаём только цену; статус "Оплачено" — в админке. |
