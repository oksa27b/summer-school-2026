## Переиспользуемые логики

Бизнес- и UI-логика, общая для нескольких экранов, вынесена в [09_Логики/](09_Логики/_INDEX.md). Описаны 8 ключевых логик: 
1. **OTP-авторизация** (вход без пароля).
2. **Расчёт доступности** (раздельные лимиты 16 чел / 12 тапок).
3. **Расчёт цены** (база + прокат).
4. **Правило 2 часов** (логика серверной отмены).
5. **Фильтрация слотов** (обработка пресетов дат).
6. **Карта маршрута** (fallback на текст в ангаре).
7. **Запрос push-разрешения** (контекстный запрос после записи).
8. **Паттерн состояний** (Loading/Error/Offline).

## Соглашения и правила

- **Платформа:** нативное мобильное приложение (iOS + Android). Используется системный push (напоминания) и хранение токенов в Keychain / Keystore.
- **Source of Truth:** Бэкенд является единственным источником истины. Все проверки лимитов (R-001) и расчет дедлайнов отмены (R-021) выполняются на сервере.
- **Дизайн-макеты:** Источник — Figma «Вертикаль приложение». Все расхождения с макетами зафиксированы в [design-review.md](../3-design-brief/design-review.md).
- **Критическое уточнение лимитов (RR-D01):** Вопреки макетам дизайна, лимит мест на одну бронь строго **не более 3** (себя + 2 гостя). ТЗ имеет приоритет над макетом.
- **Success State (RR-D03):** Экран успеха `BS-002` реализован как полноэкранный интерфейс (тип Экран), а не Bottom Sheet. ID сохранен для трассировки.
- **Offline Resilience:** Приложение поддерживает работу с кэшированными данными виджета активной брони для обеспечения прохода на ресепшн при отсутствии связи (NFR-24).
