# Описание домена и ограничения

> Этап 1. Артефакт 1 проекта: согласованное описание домена + список ограничений. 
> Заполняется после ответов заказчика ([customer-questions.md](customer-questions.md)).

## Описание домена

**Бизнес.** «Вертикаль» — скалодром, расположенный в бывшем складском ангаре, работающий третий год. Клуб проводит **групповые тренировки** по скалолазанию. Занятие длится 1.5 часа и проходит в одном из **форматов**: болдеринг с инструктажем — для новичков, трассы с верёвкой — для опытных. Каждую тренировку ведёт **инструктор** (сейчас их пятеро, в сезон нанимаются подработчики).

**Процесс.** Владелец заранее формирует **расписание на неделю вперёд**: для каждой тренировки задаются время начала, зона/формат, назначенный инструктор и число мест. Клуб располагает **12 комплектами снаряжения** (скальники + страховочная система); для новичковых групп (болдеринг) берут не более 8 человек (безопасность), для опытных — до 16 (физическая вместимость зоны). Клиент видит свободные **слоты** тренировок и **записывается**, указывая, пойдет он со своим снаряжением или возьмёт его **напрокат**. До начала клиент может **отменить** запись — при ранней отмене место освобождается, поздняя отмена оставляет место в группе и снаряжение простаивающими. Сейчас запись ведётся вручную (Telegram + тетрадь), что приводит к двойным записям и путанице.

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на тренировки (просмотр и фильтрация слотов, бронирование, выбор снаряжения, отмена, свои бронирования), исключив ручную путаницу. Управление расписанием и работу персонала обеспечивает **существующая инфраструктура**; приложение потребляет её данные через API.

**Ключевые объекты домена:**

- **Тренировка / слот** — конкретный сеанс в расписании: время начала, формат, инструктор, всего и свободно мест. Создаётся в существующей инфраструктуре.
- **Формат / Зона** — вариант тренировки (болдеринг / верёвка), определяет сложность и потолок мест (справочные данные, read-only).
- **Инструктор** — ведёт тренировку; для клиента — справочные данные (read-only).
- **Снаряжение** — инвентарь (12 комплектов: обувь + система); может выдаваться напрокат.
- **Запись (бронь)** — резерв места клиентом на конкретный слот: число мест, своё/прокатное снаряжение, статус.
- **Клиент** — записывается на тренировки, управляет своими записями.

> Роли Инструктор и Владелец/Администратор обслуживаются **существующей инфраструктурой** и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Посетитель скалодрома. Пользуется приложением с телефона. | Увидеть свободные тренировки, записаться (себя и гостей), выбрать прокат снаряжения, видеть свои брони, отменить запись. | **Да** |

## Ограничения

**Бизнес-ограничения**

- Всего **12 комплектов снаряжения** — общий потолок прокатного фонда клуба.
- Новичковая группа — **не более 8 человек**; опытная — до 16.
- Лимит мест слота = `min(свободное снаряжение, потолок формата)`.
- **Политика отмен:** бесплатная отмена не позднее чем за **2 часа** до старта; поздняя отмена (<2 ч) фиксируется без освобождения места и без штрафов.
- **Оплата офлайн** (наличные / перевод в зале) — приложение лишь фиксирует запись и показывает цену.
- **Период списка слотов по умолчанию — 7 дней (R-027).** Клиенту показываются слоты на ближайшую неделю. Более длинный период доступен через фильтр дат. При отсутствии расписания показывается **empty state** («Пока нет доступных тренировок»).
- **Отмена тренировки скалодромом / профилактика (R-008).** Если тренировка отменяется (проверка снаряжения, перекрутка трасс), бронь получает статус **«Отменена скалодромом»**; клиенту приходит **push-уведомление**. Повторная запись на отменённый слот запрещена.

**Технические / платформенные**

- **Mobile-first**: использование на телефоне прямо в зале (крупные элементы для удобства работы, когда руки в магнезии).
- Работа через **API с существующим бэкендом** (Black-box источник истины).
- **Гарантия «0 двойных броней»** обеспечивается на стороне бэкенда.
- **Данные существующей инфраструктуры (R-015).** Считаем контракт API каноническим; бэкенд отдаёт все поля (цены, зоны, свободный фонд инвентаря).

**Проект**

- **Срок** — запуск к началу нового сезона (~2 месяца).
- **Бюджет** ограничен → только MVP (Minimum Viable Product).

**Границы скоупа**

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура) |
| :-- | :-- |
| Регистрация/вход клиента по телефону (OTP) | Создание и редактирование расписания |
| Просмотр и фильтрация тренировок (дата, формат) | Оценка инструктора и рейтинги (**Phase 2**) |
| Запись (до 3 мест за раз) | Отметка явки/неявки клиента инструктором |
| Выбор «своё / прокатное снаряжение» | Онлайн-эквайринг (оплата картой в приложении) |
| Список своих бронирований и их отмена | Программа лояльности и скидки |
| Push-уведомления о напоминаниях и отменах | Автоматическое закрытие зон на профилактику |

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Тренировка / слот** | Запланированное занятие: время, формат, инструктор, доступные места. |
| **Формат** | Тип лазания: болдеринг (новички, лимит 8) или высокая стена (опытные, лимит 16). |
| **Снаряжение** | Комплект (скальные туфли и страховочная система). Всего 12 комплектов. |
| **Прокат** | Аренда клубного снаряжения. Своё снаряжение не расходует лимит из 12 штук. |
| **Запись (бронь)** | Резерв места на тренировку (1–3 человека) с указанием типа снаряжения. |
| **Отмена** | Снятие записи. Ранняя (≥2 ч) — освобождает место. Поздняя (<2 ч) — место остается занятым. |
| **Отмена скалодромом** | Техническая отмена (профилактика). Клиент уведомляется через Push. |
