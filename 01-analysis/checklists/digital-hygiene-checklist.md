# Чек-лист цифровой гигиены · «Вертикаль»

> Этот документ фиксирует финальную проверку аналитических артефактов перед передачей их в разработку. Цель — исключить логические ошибки, "галлюцинации" ИИ и несоответствие брифу.

## Галлюцинации и достоверность
- [x] **Никакой онлайн-оплаты:** Проверено, что во всех ТЗ и API оплата зафиксирована как строго офлайн (в зале), согласно бюджетным ограничениям Оли.
- [x] **Яндекс.Карты:** Убедилась, что выбрана интеграция именно с Яндекс.Картами (актуально для РФ), а не выдуманные ИИ зарубежные сервисы.
- [x] **Снаряжение:** Проверено, что количество комплектов (12 шт) соответствует брифу и не "галлюцинирует" в сторону увеличения.
- [x] **Терминология:** Все сущности (Слот, Формат, Снаряжение) согласованы и внесены в глоссарий.

## Корректность требований
- [x] **Раздельные лимиты (R-001):** ФТ и НФТ четко разделяют физические места в зале (8/16) и фонд снаряжения (12). Нет логической ошибки "место = тапки".
- [x] **Правило 2 часов:** В Use Cases прописано, что тип отмены (ранняя/поздняя) определяет сервер, а не телефон клиента (Source of Truth).
- [x] **Идемпотентность (Idempotency):** Проверено, что все мутирующие запросы (запись/отмена) используют **Idempotency-Key** для защиты от дублирования броней при обрывах связи в ангаре.
- [x] **Smart UI:** Виджет активной брони явно описан как замена разделу истории для MVP.
- [x] **Проверяемость:** Каждое функциональное требование (FR) имеет критерий приемки (AC) в формате Gherkin.

## Специфика домена (Climbing Gym Context)
- [x] **Магнезия-friendly:** Проверено наличие требований к размеру тач-зон (48pt+) в Foundations и ТЗ на экраны.
- [x] **Безопасность (ТБ):** Включено обязательное требование о подтверждении техники безопасности при первой записи.
- [x] **Оффлайн-режим:** В НФТ прописано требование о кэшировании данных активной брони (виджета) для прохода на ресепшн без связи.

## Полнота и Скоуп
- [x] **Phase 2:** Рейтинги инструкторов и лист ожидания явно вынесены за границы MVP для соблюдения срока запуска (2 месяца).
- [x] **Удаление данных:** Добавлена функция удаления аккаунта (обязательно для App Store/Google Play), даже если её не было в исходном письме.
- [x] **Вопросы заказчику:** Все 19 вопросов из сессии выявления требований (Elicitation) нашли отражение в спецификациях.
