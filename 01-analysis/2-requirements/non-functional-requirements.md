# Нефункциональные требования (НФТ)

> Этап 2. Качество системы: производительность, надёжность, безопасность и удобство. 

> **Скоуп — клиентское мобильное приложение и API для него.** НФТ относятся к мобильному клиенту; административные функции обеспечиваются существующей инфраструктурой.

> **Целевая платформа.** Нативное/гибридное приложение для iOS и Android. Распространение: App Store и Google Play.

| ID | Категория | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- | :-- |
| **NFR-1** | Удобство (UX) | **Дизайн для зала:** Интерфейс mobile-first. Крупные тач-зоны (≥ 44pt) и высокий контраст. Учёт контекста: использование **руками в магнезии** и при ярком освещении зала. | Высокий | Бриф / Домен |
| **NFR-2** | Удобство (UX) | **Скорость записи:** Путь от открытия приложения до подтверждения брони — не более 3-х экранов. | Высокий | Бриф |
| **NFR-3** | Удобство (UX) | **Минимальный барьер:** Регистрация только по имени и телефону (без пароля). | Высокий | Ответ №6 |
| **NFR-4** | Совместимость | Поддержка актуальных ОС: iOS 15+ и Android 9+. Использование системных Push и защищённого хранилища (Keychain/Keystore). | Must | R-002 |
| **NFR-6** | Производительность | Время отклика API при загрузке расписания — менее 2 секунд при стабильном 4G/Wi-Fi соединении. | Высокий | M-6 |
| **NFR-8** | Надёжность | **Атомарность бронирования:** Исключение овербукинга. Количество занятых мест и выданного снаряжения никогда не должно превышать лимитов (16 чел / 12 компл). | Критичный | R-001 |
| **NFR-11** | Безопасность | Персональные данные (имя, телефон) должны передаваться в зашифрованном виде. Доступ клиента к чужим броням через API запрещён. | Высокий | NFR-20 |
| **NFR-15** | Ограничения | Срок реализации и запуска MVP — строго 2 месяца (к началу сезона). | Критичный | BR-7 |
| **NFR-17** | Уведомления | Доставка напоминаний через системный Push. Тайминг: за 24 часа и за 2 часа до начала тренировки. | Средний | FR-33 |
| **NFR-18** | Безопасность | Управление сессиями через JWT (access + refresh). Обмен данными только по протоколу TLS 1.2+. | Высокий | R-016 |
| **NFR-19** | Антифрод | Ограничение частоты запросов SMS OTP: не более 1 кода в минуту на один номер телефона. | Высокий | Бюджет |
| **NFR-21** | Метрики (p95) | 95% запросов на создание брони должны обрабатываться быстрее 1.5 сек. | Высокий | R-018 |
| **NFR-23** | Доступность | Целевая доступность сервиса — 99% в рабочее время скалодрома. | Средний | R-019 |
| **NFR-24** | **Отказоустойчивость** | 1. **Идемпотентность:** Использование `Idempotency-Key` для предотвращения дублей брони при обрыве связи. <br>2. **Offline-кэш:** Список «Мои записи» должен быть доступен без интернета (последнее успешное состояние). <br>3. **Feedback:** При потере сети приложение отображает баннер «Нет соединения». | Высокий | **Resilience Design** |
| **NFR-25** | Доступность (A11y) | Соответствие контрастности текста стандарту WCAG 2.1 AA. Поддержка системного увеличения шрифта. | Средний | R-029 |
| **NFR-26** | Интеграции | Карты — Яндекс.Карты. API-ключ в Remote Config для возможности смены без обновления приложения. | Средний | R-011 |
| **NFR-27** | **Мониторинг** | Интеграция системы сбора крашей (напр. Firebase Crashlytics) для оперативного исправления багов после запуска MVP. | Высокий | **Support Design** |

## Технический стек (Предположение)
- **Frontend:** Flutter / React Native (для кроссплатформенности и скорости).
- **Backend:** Python (FastAPI) / Go (для высокой производительности API).
- **Database:** PostgreSQL (поддержка транзакционности для исключения овербукинга).
