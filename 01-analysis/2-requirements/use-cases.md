# Use cases / сценарии (Обновлено после ревью)

> Этап 2. Сценарии использования. Описаны основной поток, альтернативные сценарии и обработка критических конфликтов.

## UC-1: Запись на тренировку (себя и гостей)

**Актёры:** Клиент.
**Предусловия:** Клиент авторизован; в расписании есть свободные тренировки.

### Основной поток:
1. Клиент открывает список тренировок.
2. Система загружает актуальные данные о слотах (места и инвентарь).
3. Клиент выбирает подходящий слот.
4. Клиент указывает количество мест (1–3).
5. Для каждого места выбирает тип снаряжения: «Своё» или «Прокат».
6. **Валидация на стороне клиента (UI):** Если выбранное кол-во проката > доступного остатка в кэше, кнопка подтверждения блокируется (disabled) с подсказкой.
7. Клиент подтверждает запись. Генерируется и передаётся `Idempotency-Key` (R-022).
8. **Система проверяет лимиты на сервере:**
   - Лимит на 3 места.
   - Вместимость зала (8/16 чел).
   - Наличие инвентаря (12 шт).
   - **Пересечение по времени:** Проверка, нет ли у клиента другой активной брони в это же время.
9. При успехе система фиксирует запись, возвращает статус `201 Created` и отображает виджет на главном экране.

### Исключения / ошибки:
- **E4: Сетевой сбой.** При повторном запросе с тем же `Idempotency-Key` сервер возвращает **первоначальный успешный ответ (201)**, не создавая дубль и не выдавая ошибку.
- **E6: Пересечение (Time Conflict).** Если у клиента уже есть бронь на это время, запись отклоняется с кодом `conflict_booking` (нельзя быть на двух тренировках сразу).

### Матрица ошибок (дополнение):

| Код | HTTP | Описание | UX-реакция |
| :-- | :-- | :-- | :-- |
| `conflict_booking` | 409 | Пересечение с другой бронью | Сообщить: «У вас уже есть запись на это время» |

## UC-2: Отмена бронирования

**Актёры:** Клиент.
**Триггер:** Клиент нажимает «Отменить» на виджете.

> **Бизнес-логика (R-021):** Источник истины — сервер. 
> - ≥ 2 ч: статус `cancelled` (ресурсы возвращаются в фонд).
> - < 2 ч: статус `late_cancel` (ресурсы заблокированы в системе до конца слота).
