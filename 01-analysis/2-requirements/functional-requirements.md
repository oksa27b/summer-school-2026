# Функциональные требования

> Этап 2. Что система должна делать (функции, поведение). 
> Обновлено после ревью: скорректирована логика лимитов и дедлайнов.

Приоритеты по методу MoSCoW: **Must** (Обязательно), **Should** (Желательно), **Could** (При наличии ресурсов), **Won't** (Вне скоупа).

## Границы скоупа (клиентское приложение + API)

Скоуп — **клиентское мобильное приложение и API для него**. Функции инструктора и администратора обеспечиваются **существующей инфраструктурой** и здесь не описываются. Бэкенд является «источником истины» для всех расчетов.

## Просмотр и фильтрация тренировок (клиент)

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| **FR-9** | Список слотов (Default) | Система показывает тренировки на ближайшие 7 дней. Занятые слоты помечаются «Мест нет» (неактивны для записи). | Must | R-027 |
| **FR-9.1**| Виджет активной брони | При наличии у клиента актуальной записи, карточка этой записи отображается над списком для быстрого доступа (Smart UI). | Must | Решение BR-7 |
| **FR-38** | Расширенная фильтрация | Пользователь может выбрать любую дату через фильтр. Система должна поддерживать запрос данных за пределами дефолтных 7 дней. | Must | R-027 |
| **FR-9a** | Карточка слота | Отображение полных параметров: время, формат, тренер, цена, свободные места и снаряжение. | Must | Ответ №5 |
| **FR-44** | Место встречи | Отображение сектора зала и карты. При отсутствии сети — показ текстового адреса (Fallback). | Must | Ревью |

## Запись на тренировку (Логика лимитов)

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| **FR-10** | Запись на слот | Возможность забронировать место в доступном слоте. | Must | Бриф |
| **FR-10.1**| Ознакомление с ТБ | Обязательное подтверждение правил техники безопасности при первой записи. | Must | Ответ №18 |
| **FR-12** | Групповая бронь | Возможность забронировать до 3 мест за одну транзакцию (себя + до 2 гостей). | Must | R-013 |
| **FR-13** | **Валидация брони** | Система разрешает бронь N мест только если: <br>1. N ≤ 3. <br>2. N ≤ (Вместимость зала - Занято мест). <br>3. ЕСЛИ выбран прокат K комплектов, то K ≤ (12 - Выдано комплектов). | Must | **Logic Correction** |
| **FR-14** | **Приоритет инвентаря** | Отсутствие проката (K=0) не блокирует запись клиентов со своим снаряжением, если в зале есть свободные места. | Must | **Logic Correction** |
| **FR-45** | Источник цены | Приложение использует `price_total` из ответа API. Клиент не производит расчет цены самостоятельно. | Must | R-005 |

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
