## 2. Критические находки (Critical Findings)

### 🔴 RR-D01. Лимит мест на бронь (Бизнес-логика)
- **Обнаружено:** В дизайне на экране оформления (SCR-004) указано: «Можно записать до 4 мест».
- **В ТЗ (FR-12):** Установлен жесткий лимит — 3 места (себя + 2 гостя).
- **Решение:** Оставить лимит **3**. Дизайнеру необходимо исправить макет. Лимит 3 обоснован пропускной способностью и требованиями безопасности (соотношение инструктор/ученики).

### 🔴 RR-D02. Соответствие требованиям Store (Compliance)
- **Обнаружено:** На экране профиля отсутствует кнопка «Удалить аккаунт».
- **В ТЗ (FR-48):** Данная функция является обязательной (Must).
- **Решение:** Добавить кнопку «Удалить аккаунт» в нижнюю часть профиля. Без этого функционала приложение не пройдет модерацию в App Store / Google Play.

---

### 🟠 RR-D03. Архитектура навигации (Success State)
- **Обнаружено:** Экран успеха отрисован как полноэкранный интерфейс.
- **В ТЗ:** Ранее описывался как Bottom Sheet (BS-002).
- **Решение:** Принять полноэкранный вариант для усиления акцента на успехе. Обновить техническую документацию (Тип: Экран).

### 🟠 RR-D04. OTP-ввод (UX)
- **Обнаружено:** Поле ввода кода — единая строка.
- **Рекомендация:** Использовать **Segmented Input** (4 отдельные ячейки). 
- **Обоснование:** Упрощает ввод «грязными руками», исключает ошибки форматирования.

## 3. Функциональные улучшения (Small Polish)

### 🟡 RR-D05. Индикация дефицита снаряжения (Smart UI)
- **Решение:** На карточке тренировки в общем списке (SCR-002) добавить иконку «👟 Проката нет», если `free_rentals = 0`. Это предотвратит лишние переходы пользователей, которым нужна аренда снаряжения.

### 🟡 RR-D06. Тактильный отклик (Haptic)
- **Решение:** Зафиксировать в требованиях срабатывание виброотклика (Haptic Feedback) в момент перехода на экран успеха. Критично для использования в шумном зале.
