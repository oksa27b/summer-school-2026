# Требования на дизайн · SCR-003 · Карточка тренировки

**ID:** SCR-003 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Актуален · **Версия:** 1.1 · **Дата:** 2026-07-06

---

## 1. Назначение и контекст
Экран предоставляет детальную информацию о выбранном слоте тренировки. Служит фильтром для отсева нецелевых бронирований (например, если атлету не подходит тренер или нет проката).

---

## 2. Навигация
- **Входящие:** Тап по карточке в общем списке [SCR-002].
- **Исходящие:** 
    - **«Записаться»** → [SCR-004] (Оформление).
    - **Карта** → [BS-004] (Интерактивная карта).
    - **Назад** → [SCR-002].

---

## 3. Компоненты и поведение

### 3.1 Раздельные лимиты (R-001)
Система отображает два независимых статуса доступности:
1. **Места:** «Свободно N из M».
2. **Снаряжение:** 
    - Если `free_gear > 0`: «👟 Прокат доступен».
    - Если `free_gear = 0`: «👟 Места только для тех, у кого своё снаряжение». Блок выделяется контрастным (но не критическим) цветом.

### 3.2 Инструктор
- **Top Badge:** Если в API для тренера передан флаг `is_top: true`, рядом с именем отображается иконка ⭐ (Звезда). Иконка должна быть заметной, но не доминирующей (Foundations §3.5).

### 3.3 Кнопка «Записаться» (Dynamic CTA)
Поведение кнопки зависит от времени и наличия мест:
1. **Запись возможна:** `current_time < slot.start_at` И `free_seats > 0`. Кнопка активна.
2. **Мест нет:** `free_seats = 0`. Кнопка заблокирована, текст «Мест нет».
3. **Занятие началось:** `current_time >= slot.start_at`. Кнопка заблокирована, текст «Занятие уже началось». (Валидация на стороне клиента, чтобы не гонять запрос 422).

## 6. Состояния экрана
- **Loading:** Скелетоны имитируют структуру карточки.
- **Race Condition:** Если лимиты изменились за время просмотра, актуализация произойдет на следующем экране [SCR-004] при его инициализации.
