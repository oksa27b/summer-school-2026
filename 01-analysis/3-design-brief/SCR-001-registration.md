# Требования на дизайн · SCR-001 · Регистрация / Вход

**ID:** SCR-001 · **Тип:** Экран · **Зона:** НЗ · **Приоритет:** Critical · **Статус:** Актуален · **Версия:** 1.1 · **Дата:** 2026-07-06

---

## 1. Назначение и контекст
Единая точка входа. Задача: максимально быстрый вход/регистрация без пароля. 
Поток: **Телефон** (Step 1) → **SMS OTP** (Step 2) → **Имя** (Step 3, только для новых).

## 2. Пользователь и цель
- **Цель:** Начать пользоваться приложением за < 30 секунд.
- **Контекст:** Берег или зал, руки в магнезии, яркий свет. Требуется поддержка автоподстановки кода и крупные тач-зоны.

## 3. Навигация
- **Входящие:** Запуск (при отсутствии токена), выход из профиля.
- **Исходящие:** Главный экран [SCR-002] после валидации кода или ввода имени.

## 4. Контент-инвентарь

| Элемент | Шаг | Назначение |
|---------|-----|------------|
| Поле «Телефон» | 1 | `contentType: telephoneNumber`. Маска +7. |
| Юридический блок| 1 | Текст + ссылки на Политику и Соглашение (Compliance). |
| Поле «Код» | 2 | `contentType: oneTimeCode`. Segmented (4-6 ячеек). |
| Иконка правки ✎ | 2 | Быстрый возврат на Шаг 1 для исправления опечатки в номере. |
| Таймер повтора | 2 | Обратный отсчёт (напр. 00:60). |
| Поле «Имя» | 3 | `capitalization: Words`. Только для новых пользователей. |

## 6. Компоненты и поведение

### 6.1 Автоподстановка (Autofill)
Поля должны поддерживать системные теги (`textContentType`):
- **Телефон:** `telephoneNumber` (предложение номера владельца над клавиатурой).
- **OTP-код:** `oneTimeCode` (подстановка кода из входящего СМС в один тап).

### 6.2 Тактильный фидбек (P4)
- **Haptic:** При заполнении последней (четвертой) ячейки кода и успешном срабатывании автоматической валидации устройство должно выдать легкий виброотклик.

### 6.3 Исправление номера (Step 2)
Рядом с номером телефона на Шаге 2 отображается иконка «карандаш» (✎). Тап по ней возвращает пользователя на Шаг 1 с фокусом на поле ввода, позволяя быстро исправить опечатку.

## 7. Состояния экрана

| Состояние | UI Реакция |
|-----------|----------------|
| **Rate Limit (429)** | Текст под полем: «Слишком много попыток. Попробуй через час». Кнопка заблокирована. |
| **Invalid Code** | Подсветка ячеек красным, текст: «Неверный код». |
| **Loading** | Скелетон или лоадер в кнопке. |

## 8. Критерии приёмки (Gherkin)

```gherkin
Сценарий: Быстрое исправление номера
  Дано: Клиент находится на Шаге 2 и ждет СМС.
  Когда: Он нажимает на иконку «карандаш» рядом с номером.
  Тогда: Система возвращает его на Шаг 1.
  И: Номер в поле уже заполнен, курсор стоит в конце строки.

Сценарий: Согласие с правилами
  Дано: Клиент на Шаге 1.
  Тогда: Он видит активные ссылки на юридические документы.
  И: Текст подтверждает, что нажатие кнопки является фактом согласия.

Сценарий: Автоподстановка кода
  Дано: Клиенту пришло СМС с кодом.
  Когда: Он тапает на код над клавиатурой.
  Тогда: Все ячейки segmented-поля заполняются автоматически.
  И: Срабатывает виброотклик.
